# Approximation Algorithms Study Notes

## Overview

Approximation algorithms are essential tools in tackling optimization problems, especially those that are NP-complete. These algorithms provide "near-optimal" solutions efficiently when finding the exact solution is computationally infeasible. The focus is on designing algorithms with predictable performance guarantees, expressed as the **approximation ratio** between the approximate and optimal solutions. In this study note, we explore the theoretical foundations, key concepts, algorithms, and important examples outlined in Chapter 35 of *Introduction to Algorithms, 3rd Edition*.

---

## Key Concepts

- **Optimization Problems**: Problems requiring the best (maximum/minimum) solution according to a given criteria.
- **Approximation Ratio**: For a maximization problem, it is the ratio \( C^* / C \), where \( C^* \) is the value of the approximate solution and \( C \) is the optimal solution. For a minimization problem, it is \( C / C^* \). Ratios are always \( \geq 1 \) for minimization and \( \leq 1 \) for maximization problems.
- **Polynomial-Time Approximation Algorithms**: Algorithms that find near-optimal solutions in polynomial time. Commonly evaluated by their approximation ratio.
- **Approximation Scheme**: An algorithm that takes a parameter \( \epsilon \) and produces a \( (1 + \epsilon) \)-approximation for an optimization problem.  

  - **PTAS (Polynomial-Time Approximation Scheme)** has runtime polynomial for fixed \( \epsilon \).  
  - **FPTAS (Fully Polynomial-Time Approximation Scheme)** has runtime polynomial in both the input size \( n \) and \( 1/\epsilon \).
- **Greedy Technique**: Often used in approximation algorithms, where a locally optimal choice is made at each step to contribute to the solution.
- **Tradeoff Between Time and Solution Quality**: Many approximation schemes offer better quality solutions at the cost of higher running time.

## Algorithms and Techniques

### APPROX-VERTEX-COVER (2-Approximation Algorithm for Vertex Cover)

The **vertex-cover problem** is an NP-complete problem where the objective is to find the smallest subset of vertices such that every edge in the graph is incident on at least one vertex in the subset. **APPROX-VERTEX-COVER** guarantees a solution no more than twice the size of the optimal vertex cover.

#### How It Works

- The algorithm iteratively selects an arbitrary edge, adds both its endpoints to the vertex cover, and removes all incident edges.
- This process ensures that all edges are covered while maintaining the \( 2 \)-approximation factor.

#### Pseudocode

```python
APPROX-VERTEX-COVER(G):
    C = {}              # Initialize the vertex cover as empty
    E' = G.E            # Clone the edge set of the input graph
    
    while E' is not empty:
        Pick an arbitrary edge (u, v) from E'
        Add u and v to C
        Remove all edges incident on u or v from E'
        
    return C
```

#### Complexity

- **Time Complexity**: \( O(V + E) \)  
  With adjacency list representation, iterating over edges and vertices is linear.
- **Space Complexity**: \( O(V + E) \), for storing the graph and intermediate data structures.

#### Theorem 35.1: Approximation Guarantee

*APPROX-VERTEX-COVER is a polynomial-time \( 2 \)-approximation algorithm.*

In summary:
1. **Vertex Cover Validity**: Every edge is covered because either endpoint of each edge in \( E \) is included in \( C \).
2. **Approximation Bound**: The algorithm's vertex set size \( |C| \) is at most \( 2 \times |C^*| \), where \( |C^*| \) is the size of the optimal cover.

#### Example Walkthrough

Referencing Figure 35.1:

- Input Graph (7 vertices, 8 edges):
  - Step 1: Select edge \( (b, c) \). Add \( b \) and \( c \) to \( C \). Remove edges incident on \( b \) and \( c \).

Solution \( C = \{b, c, d, e, f, g\} \) is twice the size of the optimal \( C = \{b, d, e\} \).

### Greedy Approximation Algorithm for Set Cover (Logarithmic Approximation)

**Set cover problem** requires finding a collection of subsets that together contain all elements, with a minimum set count.

- **Greedy Method**: Repeatedly pick the subset that covers the largest number of uncovered elements.

**Approximation Guarantee**: \( O(\log n) \)-approximation.

---

### 2-Approximation for TSP with Triangle Inequality

The **traveling salesman problem (TSP)** requires finding the shortest Hamiltonian cycle in a graph.

Approach (with triangle inequality):
1. Compute a Minimum Spanning Tree (MST).
2. Perform a preorder traversal of the MST.

**Time Complexity**: \( O(E \log V) \) due to MST computation.

**Approximation Guarantee**: Solution cost is at most twice the optimal TSP tour.

---

### FPTAS for Subset-Sum Problem

The **subset-sum problem** is where a sum close to, but not exceeding, a target value is desired.

- **Key Idea**: Dynamic programming with approximate filtering.
- **Time Complexity**: \( O((1 / \epsilon) n^2) \), where \( \epsilon \) is the error tolerance.
